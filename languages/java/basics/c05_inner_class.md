## 为什么要有内部类
单一类在真实设计中不可避免会出现“语义膨胀 + 行为分化 + 交互复杂化”，而语言需要一种“结构性拆分但不破坏整体语义”的能力。

防止设计退化


对类自身而言：解决“语义膨胀但不可随意拆分”的问题
    将逻辑拆分到多个子类中，解决语义混乱的情况

对可变与扩展场景而言：提供“结构化的交互约束”
    通过传入符合内部类结构 的 自定义部分实现，灵活性

对整体设计而言：避免语义冲突，维持模型清晰度
    设计意图通过命名、文档表达，通过顶级类实现相关功能，模型边界会模糊
