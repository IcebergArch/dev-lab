# 序列化

## why?
为什么 Java 当年一定要引入序列化？

结合 Java 的历史背景：

分布式计算兴起

RMI（远程方法调用）

Applet / 网络对象

强面向对象设计

👉 Java 的设想是：

对象不仅能在本地方法间流动，也能在网络与进程间流动。

而在强 OOP 语言中：

传“对象” ≠ 传“字段”

必须保留类型和结构

👉 序列化是 Java 对“分布式 OOP”的一次直接支持。

## 现状？
它的问题包括：
- 性能差
- 安全风险极高（反序列化漏洞）
- 版本演进困难
- 强耦合 JVM 内部实现

所以今天的结论是：👉 “序列化这个概念是必要的，但 Java 原生序列化不再推荐使用。”

现代替代方案：
- JSON / Protobuf / Avro
- 显式 DTO
- 自定义序列化协议
