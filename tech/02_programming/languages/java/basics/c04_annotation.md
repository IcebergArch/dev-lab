# annotation

注解：对原始代码的标记，该部分信息可以通过提供的注解框架获取

## 为什么要有注解

注解的引入，是为了让 Java 具备“声明式编程的基础设施”。

注解的存在，是为了让“代码如何被理解和处理”这件事，本身也成为代码的一部分，而不是散落在配置、命名和文档中。

语义信息只能靠“约定”或“硬编码”
    xml文件维护
横切关注点无法结构化表达
    事务、权限、校验、序列化、映射；没有注解就得正交，显示关联相关逻辑
框架无法“读懂”代码意图
    该不该框架管理、要不要框架管理；需要通过更高成本的方式理解分离的代码和配置信息

没有注解：
```text
在注解之前，Java 的“声明”只能：

写在 XML

写在命名规范

写在代码逻辑里

这些方式的问题是：

不可组合

不可校验

不可反射

与代码结构割裂
```

## 注解组合

注解负责声明
执行放在：
- AOP
- 反射
- 代理
- 框架运行时

## 基础注解

@Target修饰的对象范围
@Retention定义被保留的时间长短，source-源文件保留, class-类文件保留, runtime-运行时保留
@Documented描述-javadoc，工具稳当生成使用
@Inherited阐述了某个被标注的类型是被继承的，注解可以被继承到子类
